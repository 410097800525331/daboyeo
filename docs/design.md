# DABOYEO 디자인 기준 및 AI 로딩 모션 시안

## 기준 화면

- 대상 화면: `frontend/src/pages/daboyeoAi.html`
- 대상 스타일: `frontend/src/css/daboyeoAi.css`, `frontend/src/css/common.css`
- 적용 대상: 포스터와 장르 선택 후 `분석하기` 버튼을 눌렀을 때 나오는 AI 분석 로딩 화면
- 현재 로딩 구조: `.ai-loading-card` 안에 원형 스피너, 제목, 설명 문구가 들어가는 단일 중앙 카드
- 이번 문서 목적: 실제 페이지의 색, 밀도, 반투명 질감에 맞는 모션그래픽 시안 기준을 고정하고, 바로 적용하지 않은 상태에서 후보 5개를 비교한다.

## 사이트 디자인 토큰

### 배경

- 기본 배경은 검정에 가까운 영화관 톤이다.
- AI 페이지는 `AIbackgroundImg.jpg` 위에 어두운 남색, 보라, 청색이 낮게 깔린 분위기를 쓴다.
- 배경은 화면 전체를 채우되 UI보다 튀면 안 된다.
- 새 로딩 모션은 배경을 새로 지배하지 말고 기존 페이지 배경 위에 올라가는 레이어처럼 보여야 한다.

### 색상

- 주 배경: `#000`, `#080a14`, `#0b1020`, `#111`
- 기본 텍스트: `#fff`, 보조 텍스트는 `rgba(255, 255, 255, 0.55)` 전후
- 브랜드 보라: `#7264E9`, `#9B92FA`, `#A78BFA`
- CTA 보라: `#A797FF`, `#AEA1FF`
- 선택/분석 보조광: `#7DD3FC`, `#5EEAD4`
- 보조 핑크: `#F472B6`, `#FAA8C4`
- 경고/대체 추천 강조: `#FBBF24` 계열, 로딩 화면에서는 많이 쓰지 않는다.

### 표면과 선

- 주요 표면은 반투명 글래스다.
- 기준 표면: `rgba(255, 255, 255, 0.08)` 전후
- 기준 선: `rgba(255, 255, 255, 0.10)`에서 `rgba(205, 196, 255, 0.16)` 사이
- 선택 상태는 보라 또는 청록 글로우를 얇게 사용한다.
- 강한 네온 HUD, 굵은 외곽선, 원색 레이저는 피한다.

### 형태

- 로딩 중앙 패널은 현재처럼 `width: min(640px, 100%)` 근처가 기본이다.
- 모션그래픽이 필요하면 최대 `860px` 정도까지 넓힐 수 있지만, 화면 전체를 덮는 새 페이지처럼 만들지 않는다.
- 기본 패널 반경은 `8px`.
- 포스터 프레임과 결과 카드 미리보기는 기존 AI 결과 카드 문법을 따를 때 `12px`에서 `18px`까지 허용한다.
- pill 텍스트는 기존처럼 999px 반경을 사용할 수 있다.

### 타이포그래피

- 폰트 흐름: `Pretendard`, 시스템 산세리프
- 로딩 제목: 34px에서 42px, 굵기 800-900
- 보조 설명: 14px에서 16px, 줄간 1.6-1.7
- 큰 영웅 카피처럼 만들지 않는다. 로딩은 진행 상태 UI다.

### 움직임

- 3초에서 8초 사이로 자연스럽게 반복되어야 한다.
- 실제 분석 완료 시점과 정확히 맞는 퍼센트는 불가능하므로, 숫자 진행률은 `예상 진행` 또는 단계형 상태로만 다룬다.
- 움직임은 `포스터 취향`, `장르 매칭`, `상영 후보 비교`, `추천 카드 조립` 중 하나 이상을 보여줘야 한다.
- 좌석 분석을 실제로 하지 않는다면 좌석을 핵심 그래픽으로 쓰지 않는다.

## 피해야 할 방향

- 별도 영상 템플릿처럼 보이는 과한 모션그래픽
- 실제 페이지와 다른 주황/갈색/베이지/밝은 회색 중심 팔레트
- SF 레이더, 과한 원형 HUD, 과한 네온 라인
- 영화 포스터 원본처럼 보이는 저작권 이미지
- 결과를 이미 확정한 것처럼 보이는 문구
- “좌석 분석 중”처럼 실제 추천 로직보다 과장된 표현

## 시안 1. Poster Taste Scan

### 의도

사용자가 고른 포스터 취향을 읽고 있다는 점을 가장 직접적으로 보여준다.

### 정지 이미지

- 중앙 글래스 패널 안에 포스터 카드 5장이 낮은 원근감으로 놓인다.
- 선택된 카드 2-3장에 얇은 청록 라인이 지나간다.
- 카드 아래에는 `포스터 취향 분석 중` 상태 문구와 작은 장르 칩을 둔다.

### 모션 계획

- 0-1.5초: 포스터 카드가 살짝 떠오르며 정렬된다.
- 1.5-4초: 세로 스캔 라인이 카드 위를 천천히 통과한다.
- 4-6초: 추출된 장르 칩이 하단에 한 개씩 추가된다.
- 6초 이후: 카드 흔들림 없이 스캔 라인과 칩 하이라이트만 반복한다.

### 장점

- 현재 AI 페이지의 포스터 선택 흐름과 가장 잘 이어진다.
- 구현 난도가 낮고 로딩 화면으로 자연스럽다.

### 리스크

- 단독으로 쓰면 평범한 포스터 캐러셀처럼 보일 수 있다.

## 시안 2. Recommendation Assembly

### 의도

추천 결과가 만들어지는 과정을 미리 보여준다. 실제 결과 화면으로 넘어가는 연결이 가장 좋다.

### 정지 이미지

- 중앙에 추천 카드 3개가 반투명 상태로 배치된다.
- 각 카드에는 포스터 슬롯, 점수 배지, 장르 줄, 이유 문장 스켈레톤이 순서대로 보인다.
- 상단 문구는 `추천 카드를 조립 중`으로 둔다.

### 모션 계획

- 0-1초: 빈 카드 3장이 나타난다.
- 1-3초: 포스터 슬롯과 랭크 배지가 먼저 채워진다.
- 3-5초: 장르 칩과 이유 문장 스켈레톤이 왼쪽에서 오른쪽으로 차오른다.
- 5-7초: 카드 한 장씩 보라 글로우가 지나가며 완료감을 준다.

### 장점

- “AI 분석 후 추천 결과가 나온다”는 의미가 가장 선명하다.
- 지금 결과 카드 디자인과 자연스럽게 이어진다.

### 리스크

- 실제 결과와 너무 비슷하게 만들면 사용자가 이미 결과를 받은 것으로 오해할 수 있다.

## 시안 3. Showtime Candidate Matrix

### 의도

상영 후보를 비교한다는 서비스 특성을 보여준다.

### 정지 이미지

- 영화관, 시간, 거리, 가격, 잔여 정보처럼 보이는 작은 셀 그리드를 둔다.
- 일부 셀만 보라/청록 라이트로 점멸한다.
- 포스터 미니카드 1개가 왼쪽에 있고, 오른쪽에는 후보 행렬이 정리된다.

### 모션 계획

- 0-2초: 후보 셀이 어두운 상태로 나타난다.
- 2-4초: 열 단위로 얇은 스윕 라이트가 지나간다.
- 4-6초: 적합한 후보 3개만 남고 나머지는 흐려진다.
- 6초 이후: 남은 후보 셀의 테두리만 조용히 반복 점멸한다.

### 장점

- 포트폴리오에서 “실제 상영 데이터 기반 추천”이라는 인상을 준다.
- 단순 AI 로딩보다 서비스의 계산 과정을 보여준다.

### 리스크

- 정보량이 많아지면 로딩 화면이 복잡해진다.

## 시안 4. Genre Signal Ribbon

### 의도

장르 선택값이 추천 기준으로 변환되는 과정을 표현한다.

### 정지 이미지

- `스릴러`, `액션`, `애니`, `로맨스` 같은 장르 칩이 얇은 리본 라인에 연결된다.
- 리본은 중앙의 작은 분석 코어를 지나 추천 카드 쪽으로 흐른다.
- 전체 톤은 보라 중심, 청록은 선택 완료 표시로만 쓴다.

### 모션 계획

- 0-1초: 장르 칩이 사용자가 선택한 순서처럼 진입한다.
- 1-3초: 칩에서 얇은 선이 중앙으로 연결된다.
- 3-5초: 중앙 코어가 한 번만 밝아지고 카드 방향으로 신호가 흐른다.
- 5초 이후: 리본 라인의 미세한 흐름만 반복한다.

### 장점

- 장르 선택 이후 로딩 화면으로 의미가 분명하다.
- 레이더보다 덜 유치하고 페이지 톤에 맞추기 쉽다.

### 리스크

- 포스터 기반 추천 느낌은 약하므로 포스터 슬롯을 작게라도 같이 두는 편이 좋다.

## 시안 5. Cinematic Result Reveal

### 의도

영화 포스터가 암실에서 현상되듯 결과가 서서히 드러나는 느낌을 준다.

### 정지 이미지

- 어두운 중앙 패널 안에 포스터 실루엣 3장이 흐릿하지 않은 추상 블록으로 놓인다.
- 위쪽에는 얇은 보라 라이트, 아래쪽에는 `추천 결과를 정리 중` 문구를 둔다.
- 실제 영화 포스터 대신 추상 색면만 사용한다.

### 모션 계획

- 0-2초: 어두운 포스터 실루엣이 희미하게 나타난다.
- 2-4초: 보라 라이트가 위에서 아래로 지나가며 카드 형태가 선명해진다.
- 4-6초: 카드 하단에 제목/장르/시간 스켈레톤이 드러난다.
- 6초 이후: 라이트가 약하게 다시 지나가며 반복한다.

### 장점

- 영화 서비스 분위기와 가장 잘 맞는 감성형 로딩이다.
- 과한 기능 설명 없이 고급스럽게 만들기 쉽다.

### 리스크

- 기능 설명력은 2번과 3번보다 약하다.

## 추천 우선순위

1. Recommendation Assembly
2. Poster Taste Scan
3. Showtime Candidate Matrix
4. Genre Signal Ribbon
5. Cinematic Result Reveal

최종 구현은 `Recommendation Assembly`를 중심으로 잡고, 초반 1-2초에 `Poster Taste Scan`을 섞는 방향이 가장 안정적이다. 사용자가 누른 `분석하기`의 결과가 추천 카드로 이어지는 흐름이 보이고, 현재 다보여 AI 결과 카드 디자인과도 가장 잘 붙는다.

## 이미지 생성 기준

- 모든 시안은 실제 페이지처럼 어두운 영화 배경 위의 중앙 로딩 패널로 구성한다.
- 브랜드 텍스트는 `DABOYEO`만 작게 사용한다.
- 실제 영화 포스터, 실제 배우 얼굴, 실제 영화 로고는 쓰지 않는다.
- 텍스트는 한국어로 짧게 둔다.
- 시안 이미지는 구현 산출물이 아니라 방향 검토용이다.
